# Colors

Our color system helps users identify status, see actions, locate help, and indicate next steps. The consistent use of color keeps cognitive loads low and creates a unified, engaging user experience. Colors are designed to be clear and accessible.

---

## Color Keys

Colors are organized into **Primitive Colors** (raw color values) and **Semantic Colors** (purpose-driven tokens). Components should always use semantic colors to ensure consistency and maintainability.

## Usage in Code

### CSS Variables

All colors are available as CSS custom properties:

```css
/* Semantic colors */
.my-element {
  color: var(--semantic-text-primary);
  background-color: var(--semantic-bg-primary);
  border-color: var(--semantic-border-layout);
}

/* Primitive colors */
.my-element {
  color: var(--color-primary-500);
  background-color: var(--color-neutral-100);
}
```

### TypeScript

Import color tokens from the colors module:

```typescript
import { semantic, primitives, primary, secondary } from '@/lib/colors'

// Using semantic colors
const primaryColor = semantic.primary.color      // #343E55
const brandColor = semantic.brand.color          // #2BBCCA
const textColor = semantic.text.primary          // #181D27

// Using primitive colors
const blue500 = primary[500]                     // #343E55
const gray100 = primitives.neutral[100]          // #F5F5F5
```

### Tailwind Classes

Use the defined color classes in your components:

```tsx
// Using Tailwind with semantic variable classes
<button className="bg-[var(--semantic-primary)] text-[var(--semantic-text-inverted)]">
  Click me
</button>

// Using hardcoded semantic values (for components)
<button className="bg-[#343E55] text-white hover:bg-[#2F384D]">
  Click me
</button>
```
